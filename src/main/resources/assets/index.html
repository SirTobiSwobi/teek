C3 Taxonomy Extension system for Emerging Knowledge (TEEK)

This microservice implements a subset of the C3 Classifier-Trainer API. It uses the Word2Vec-Plus model and builds on it to create suggestions for sub-categories in existing taxonomies. The taxonomies are edited using the /categories and /relationships endpoints. The suggestions are either generated using the Arithmetic Vector Stream Predictor (AVSP) or the Regression Vector Stream Predictor (RVSP). Which one is used is specified in the active model which encapsulates configuration objects. The mechanism to generate and edit both is equal to that of the C3 Classifier-Trainer API. Additional Endpoints under the categories endpoint provide the generated predictions.

/metadata

Renders the metadata returned by the service.

/documents

Here, all documents relevant for the classifier trainer can be created, read, updated and deleted.

/categories

Here, all categories relevant for the classifier trainer can be created, read, updated and deleted.

/relationships

Categories can form relationships. This means, that certain categories can be sub-categories of others. They also can be equivalent to each other. This has logical implications for assigned documents. Documents assigned to sub-categories are always implicitly also assigned to their super categories as well as to their equivalent categories. The service automatically computes these relationships. Here you can edit these relationships.

/configuration

Depending on the utilized machine learning algorithm, different hyper-parameters for the learning algorithm can be specified. This happens using this API endpoint.

/models

Models are generated, when the trainer learns a representation from the existing examples. These models can then be shared with other athlete services that implement the same model. Generating models triggers training,

/model

Every trainer service can also serve as athlete service. Every athlete service needs to select one active model that is used for every categorization.

/categories/$c/suggestions/$m

This GUI element can be used to generate suggestions for categories and relationships.

/wordembeddings

Wordembeddings are automatically generated representations of semantic meanings of words. These are usually stored in large files. The endpoint allows uploading these wordembeddings to use in Text Categorization.

Taxonomy Extension for Emerging Knowledge

This microservice generates suggestions to extend existing taxonomies by employing machine learning. TEEK heavily relies on provided word embeddings. It generates it's suggestions based on the word embedding. Different word embeddings generate vastly different results. A Trainer Service always includes an Athlete Service. Each C3 Service is available as JAR file and Docker Container. This makes them runable anywhere.
The service employs multiple endpoints using a REST/JSON API. This is wrapped in the c3.ct.trainer-api.js JavaScripts employed by this web GUI.
